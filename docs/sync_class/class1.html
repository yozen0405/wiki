第一章 程式語言初探

1.1 程式語言是什麼

程式語言是一種人與電腦溝通的工具，這種允許了工程師能將想法和指令轉化為電腦能夠理解和執行的形式。
每種程式語言都有其特定的語法和規則，並且可以應用於不同的應用領域。程式語言目前的應用，包括軟體應用程式、網站、遊戲、大數據分析、AI 等。

1.1.1 語言的種類

我們注要會分成 Python 一類，C, C++, Java 一類，Python 的特點是語法簡潔，好寫，但效率會比較差，注意在寫一些簡單的程式會很好用，C, C++, Java 則是語法比較困難，但能讓使用者更加細節的運用其中的語法，在寫一些大型專案時較有優勢。

其實程式語言的概念都是差不多的，而在學完 C, C++ 後再來學 Python 也能非常容易上手。我們這堂課主要會學習的就是 C++。

1.3 整合式開發環境 IDE

1.3.1 介紹

在寫好程式後，我們會需要一個編譯器(compiler)能讓電腦接收到我們所輸入的訊息，就像翻譯師，把我們寫的程式碼轉換成電腦能夠理解的語言，讓它讀懂並執行我們的代碼。

而整合式開發環境（IDE）則是一個集成了多個功能的環境，其中包括編譯器。

下面是我有用過的兩個編譯器

  • Vscode: 功能最強大，是工程師一般都會使用的，但需要額外安裝一些插件，對新手稍不友善
  • Code::blocks: 適合新手上手，但對於工程師功能相對比較不好調整。

我們這邊會使用很好上手的 Code::blocks，以下是安裝教學。

1.4 第一支程式

1.4.1 Hello World!

試著在 CodeBlocks 輸入以下代碼:

#include <iostream>
using namespace std;

int main() {
    cout << "hello world!";
		return 0;
}

剛剛的程式，大概可以分為以下部分:

包含函數

有點像工具包，像這個程式中我們要輸出東西到螢幕上，要用 cout 這個函數，而 cout 是在 <iostream>（Iput/Output Stream，掌管輸入輸出）這個函數庫中。而 < > 則是包住標頭檔的符號。

命名空間

有點像是工具欄，為了避免函數混淆而需要聲明函數所在的位置，因為可能有很多地方都有相同的函數。當你使用 using namespace std 時，你可以直接使用 std 命名空間中的元素，而不需要在每個元素前加上 std::。例如 std::cout 你就可以直接像上面一樣寫 cout。

主函數

程式的入口，也就是我們想要電腦做的指令。在函數的後方我們會打上一個括號。在函數所包含的區塊兩側，我們則會打上一個左大括號與一個右大括號。在主函數中，程式會由上而下一行一行的被運行。

我們來解析裡面的 code，cout 的部分我們在下面 1.5. C++ 的輸出 會提到，通常使用 return 0 來表示程式成功執行並正常退出。一般來講，返回 0 通常表示程式成功執行，而其他非零值則表示程式發生了錯誤或異常情況。

在 C++ 的程式中，每個語句都應該以分號做結尾，讓電腦知道這是一行的結束。注意不要打成冒號。而在函數宣告和包含函數，是不需要加上分號的。

1.5 C++ 的輸出

1.5.1. 基礎輸出

在 C++中，我們使用 cout 進行基礎的文字輸出，後面記得加上 << ，代表著將後面的字串傳給 cout。而 endl 是換行的意思。

#include <iostream>
using namespace std;

int main() {
    cout << "Hello, World!" << endl;
    return 0;
}

也可以進行多筆輸出，如下:

#include <iostream>
using namespace std;

int main() {
    cout << "Line 1";
    cout << "Line 2";
    cout << "Line 3";
    cout << "Line 4";
    return 0;
}

或者

#include <iostream>
using namespace std;

int main() {
    cout << "Line 1" << "Line 2" << "Line 3" << "Line 4";
    return 0;
}

1.5.1.2. 引號的意義

在文字中，我們使用單引號 ' ' 或雙引號 " " 來分別表示字元(char)跟字串(string)。單引號用於表示單一字元，例如 'A' 或 '5'。雙引號用於表示字串，例如 "Hello, World!", "apple"。

1.6. 註解

在 C++ 中，註解(comments)是寫給人看的，我們一般會在需要使用註解那行前面加上 //，或是如果想要包住好幾行可以用 /*, */。主要是讓其他人能讀懂我們的 code，電腦在編譯的時候會忽略。

軟體工程師在寫程式的時候常常會寫一些註解，除了在寫 code 的思緒會比較清楚外，對於其他人在閱讀時也更容易理解。

1.7. 補充： C++ 的 code format

1.7.1. 什麼是 code format(代碼格式)？

code format 分兩個部分: 縮排(Indentation)與空格(spacing)。縮排是指將程式碼的每一層結構向右移動一點，以使程式的邏輯結構更加清晰易讀，而空格也是差不多的概念。

1.7.2. 為什麼需要縮排和空格？

縮排有助於區分不同的程式區塊，使程式的邏輯結構更容易理解。就像數學公式中，括號的使用可以幫助我們理解運算優先順序一樣。好的程式縮排也有助於你抓錯，別人看的時候也會更容易理解。

1.7.3. 如何保有好的 code format？

在 C++ 中

  1. 通常使用 Tab 進行縮排。也就是在每一行前面加上一個 Tab，例如剛剛的 cout 之前。
  1. 在操作符號周圍留有空格，例如剛剛的 cout 與 << 之間我們加上空格。

在 Google 的標準程式碼風格就是 K&D，有興趣的可以上網搜尋。其實不用特意學，最好的方式就是參考別人的程式，進而從中學習即可。
